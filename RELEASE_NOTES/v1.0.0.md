Erste öffentliche Version des Checkmk Cockpit — Windows-Desktop-Tool für die
täglichen Handgriffe an einem **Checkmk 2.5** über die REST-API v1, gebaut für
den Fachbereich **5424 IT-Basis-Dienste**.

## Highlights

**Status-Tab**
- Live-Status aller Hosts und Services, Ampel-Anzeige, Auto-Refresh
- Freitext-Filter + „Nur Probleme"
- **Ack + Downtime** direkt aus der Liste (Toolbar oder Rechtsklick), mit
  Dauer-Presets (1h / 2h / 4h / bis morgen 06:00) und Pflicht-Kommentar
- **Bulk-Ack/Downtime** per Ctrl-/Shift-Klick: ein Kommentar für alle,
  Fortschrittsanzeige „Ack 3/12: host/service", Einzelfehler brechen den
  Bulk nicht ab

**Host-Details (Doppelklick oder Rechtsklick)**
- Eigenes Fenster mit Host-State, Ordner/IP/Alias/Plugin-Output
- Alle Services des Hosts inkl. Aggregat-Zählern
- Ack und Downtime auch auf **kompletten Host** („ganzer Host in Wartung")
- Mehrere Detail-Fenster parallel möglich

**Host-Filter mit Favoriten**
- Persistente benannte Filter (Combobox in Status- und Konfig-Tab)
- Zwei Modi je Favorit: **Hostname-Regex** oder **explizite Include-Liste**
- Aus dem Konfig-Tab: Ctrl-Klick auf mehrere Hosts → „Auswahl als Favorit…"
- Verwaltungs-Dialog für Anlegen/Bearbeiten/Löschen

**Konfiguration**
- Host anlegen mit Ordner-ID-Pfad (nicht Titel!), IP, Alias
- **Service Discovery** (Toolbar-Button oder Rechtsklick): startet
  `fix_all` als Hintergrund-Task, pollt bis fertig, aktiviert die
  Änderungen — bringt bestehende Hosts wie `DBSQL01` mit einem Klick
  ins Monitoring
- „Änderungen aktivieren" mit `If-Match: *`

**Zentrale Verbindung**
- Windows: verschlüsselt auf `\\Samba01\542$\Checkmk\settings.json`
  (AES-GCM mit im Binary abgeleitetem Schlüssel — mehrere Windows-Clients
  teilen dieselbe Datei ohne Klartext auf dem Share)
- Linux: user-lokal in `~/.config/Kroste/Checkmk/settings.json` mit
  AES-GCM (machine-id-basierter Schlüssel)
- Pfad in `%APPDATA%\Kroste\Checkmk\bootstrap.json` überschreibbar

**Autoupdater (Phase 1)**
- Beim Start Prüfung gegen GitHub Releases
- Statusleisten-Badge „Update auf x.y.z verfügbar" bei neuerer Version
- Dialog mit Release-Notes und „Release-Seite öffnen" / „Später" /
  „Diese Version überspringen"
- Kein Selbst-Ersetzen des Binary — Phase 2 kommt später

**Cross-Platform**
- Windows: self-contained Single-File-EXE (kein .NET-Runtime nötig)
- Linux: tar.gz + AppImage

## Downloads

- `Checkmk-1.0.0-win-x64.zip` — Windows (empfohlen)
- `Checkmk-1.0.0-linux-x64.tar.gz` — Linux tar-Archiv
- `Checkmk-1.0.0-x86_64.AppImage` — Linux AppImage

## Was ist bewusst *nicht* enthalten

- Kein Ersatz fürs Webinterface (Rollen, Regeln, Notifications, Reports)
- Kein automatisches Selbst-Update — nur der Hinweis auf neue Versionen
- Kommentare-Feature und DB-Health-Board sind Roadmap für spätere Releases
