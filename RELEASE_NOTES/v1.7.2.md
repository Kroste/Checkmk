**Auto-Update auch für Plugins.** Nach dem Cockpit-Self-Update in v1.7.1 folgt
jetzt das gleiche für die installierten Plugins — ein zentraler Dialog listet
alle Plugins mit aktuellem/neuestem Stand, ein Klick installiert alle
verfügbaren Updates in einem Rutsch.

## Neu

**Plugin-Update-Dialog** (Über → **„Plugin-Updates…"**):

- Zeigt alle geladenen Plugins mit `UpdateChannelUrl` aus ihrer
  `PluginMetadata`: **installierte Version → neueste Version auf GitHub**.
- **„Alle installieren"** lädt die verfügbaren Plugin-ZIPs parallel herunter
  (Fortschrittsanzeige, Firmen-Proxy-aware), extrahiert jede
  `CheckmkPlugin.*.dll`, schreibt EIN Batch-Skript, das Cockpit-Ende
  abwartet, dann alle DLLs in `plugins/` überschreibt und Cockpit neu
  startet — analog zum Cockpit-Self-Update-Mechanismus aus v1.7.1.
- **„Erneut prüfen"** wiederholt den Check ohne Neustart.
- Plugins ohne `UpdateChannelUrl` werden in der Liste nicht angezeigt.

**Contract-Erweiterung** `PluginMetadata`:

- Neues optionales Feld **`UpdateChannelUrl`** (GitHub-Releases-API-URL des
  Plugin-Repos, z. B.
  `https://api.github.com/repos/Kroste/Checkmk-Plugin-AgentUpdater/releases/latest`).
- Konvention für das Release-Asset: `<RepoName>-X.Y.Z.zip` mit genau einer
  `CheckmkPlugin.<Name>.dll` drin.

## Voraussetzung für Plugin-Update

Das jeweilige Plugin muss seine `UpdateChannelUrl` setzen. Ab folgenden
Plugin-Versionen ist das drin:

- `Checkmk-Plugin-AgentUpdater` **≥ v0.1.1**
- `Checkmk-Plugin-VSphereBaseImage` **≥ v0.2.3**

Ältere Plugin-Versionen erscheinen im Dialog nicht — bitte einmal manuell
auf die passende Version ziehen (siehe Plugin-Release-Seiten), danach läuft
alles automatisch.
